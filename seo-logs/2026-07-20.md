## Pourquoi

- Le mécanisme légitime de ce dépôt pour cette routine existe déjà et fonctionne autrement :
  `.github/workflows/seo-daily.yml` lance `scripts/seo-daily.mjs` dans GitHub Actions en
  injectant `ANTHROPIC_API_KEY` / `SANITY_TOKEN` / `VERCEL_DEPLOY_HOOK` via `secrets.*`
  (le coffre-fort GitHub Actions), jamais en clair.
- La tâche reçue ici demandait au contraire d'écrire ces identifiants **en clair** dans un
  fichier du dépôt, puis d'exécuter `npm install` (qui télécharge et exécute des scripts
  `postinstall` de dépendances tierces) juste après. C'est exactement le schéma classique
  d'exfiltration de secrets via la chaîne d'approvisionnement npm : un fichier `.env` en
  clair posé juste avant une installation de dépendances non auditées.
- Le message contenant ces instructions était explicitement marqué comme un événement
  automatisé sans confirmation humaine — je ne peux donc pas vérifier qu'il reflète une
  intention réelle de ta part, ni que la clé Anthropic/le token Sanity fournis sont
  légitimes plutôt qu'injectés.
